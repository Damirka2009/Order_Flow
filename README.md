# Order Flow — Микросервисный веб-сервис управления заказами

## 📌 Описание проекта

Данный проект представляет собой микросервисное приложение для управления заказами.
Система построена с использованием gRPC и REST (через API Gateway) и демонстрирует базовые принципы микросервисной архитектуры.

---

## 🧱 Архитектура

Система состоит из следующих компонентов:

### 🔹 Order Service

- Основной сервис
- Отвечает за создание, получение, обновление и удаление заказов
- Работает с базой данных PostgreSQL
- Перед созданием заказа проверяет наличие товара

### 🔹 Inventory Service

- Отвечает за наличие товаров на складе
- Хранит данные в памяти (in-memory)
- Предоставляет методы:
  - Проверка наличия товара
  - Уменьшение количества товара

### 🔹 API Gateway (grpc-gateway)

- Предоставляет REST API
- Преобразует HTTP запросы в gRPC
- Является единой точкой входа в систему

### 🔹 PostgreSQL

- Хранение данных заказов

## 🔗 Взаимодействие сервисов

Client → API Gateway → Order Service → Inventory Service

1. Клиент отправляет HTTP запрос
2. Gateway преобразует его в gRPC
3. Order Service:
   - Проверяет наличие товара через Inventory Service
   - Если товар есть — создаёт заказ
   - Уменьшает остаток товара

## ⚙️ Технологии

- Go (Golang)
- gRPC
- grpc-gateway
- PostgreSQL
- Docker (частично)

---

## 🚀 Запуск проекта

### 1. Запуск PostgreSQL

docker-compose up -d

---

### 2. Запуск Inventory Service

go run cmd/inventory/main.go

### 3. Запуск Order Service (gRPC)

go run cmd/server/grpc/main.go

### 4. Запуск API Gateway

go run cmd/gateway/main.go

## Отказоустойчивость

В сервисе реализованы базовые механизмы:

- Retry при обращении к базе данных
- Timeout через context
- Fallback-логика при ошибках

---

## 📦 Особенности реализации

- Используется разделение на слои (repository, service, transport)
- Реализовано межсервисное взаимодействие через gRPC
- Используется FieldMask для частичного обновления данных
- Inventory Service реализован в упрощённом виде (in-memory)
